📚 Projeto de Conclusão de Curso Desenvolvido como requisito para conclusão do curso de Desenvolvimento de Sistemas no SENAI Anchieta, em dezembro de 2025.

O Inpark é um sistema desenvolvido com os seguintes objetivos:

🏢 Gerenciar estacionamentos de forma eficiente

📍 Exibir informações relevantes para motoristas próximos

🚘 Informar disponibilidade de vagas em tempo real

📊 Apresentar dados como nome, localização e demais detalhes do estacionamento

Em resumo a proposta é facilitar tanto a administração dos estacionamentos quanto a experiência dos motoristas que buscam vagas com rapidez e praticidade.

🚀 Evolução do Projeto

O que começou como parte de um Trabalho de Conclusão de Curso se transformou em algo maior. Hoje, o Inpark é um projeto para aprimorar nossas habilidades como programadores, parte do nosso portfólio profissional e um sistema em constante evolução; nosso objetivo no front-end do sistema atualmente é melhorar a experiência do usuário (UX), aprimorar a responsividade e desempenho da interface, refatorar componentes existentes, adicionar novas funcionalidades visuais e administrativas.

⚠️ **IMPORTANTE
Este repositório contém apenas o front-end (interface do sistema). O back-end, responsável pela lógica de negócio e gerenciamento de dados, está disponível em um repositório separado.**

**ENGLISH VERSION**

📚 Course Completion Project
Developed as a requirement to complete the Systems Development course at SENAI Anchieta, in December 2025.

The Inpark system was developed with the following objectives:

🏢 Efficiently manage parking lots

📍 Display relevant information to nearby drivers

🚘 Provide real-time parking space availability

📊 Present data such as parking lot name, location, and other relevant details

In summary, the proposal is to facilitate both the management of parking lots and the experience of drivers who are looking for spaces quickly and conveniently.

🚀 Project Evolution

What began as part of a Course Completion Project has grown into something larger. Today, Inpark is a project to improve our skills as programmers, part of our professional portfolio, and a system in constant evolution; our current objectives for the front-end of the system are to improve the user experience (UX), enhance the interface’s responsiveness and performance, refactor existing components, and add new visual and administrative features.

⚠️ IMPORTANT
This repository contains only the front-end (system interface). The back-end, responsible for business logic and data management, is available in a separate repository.
